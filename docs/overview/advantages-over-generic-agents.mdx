---
title: 相对于通用 Agent
description: 相对于使用通用编程Agent，使用Locus的优势
---

Locus 不仅是对 LLM API 的简单封装，为了最大化 Agent 的能力，我们做出了很多努力。

- 我们在 Unity 编辑器内实现了 C# 代码的 JIT 执行能力。基于这一能力，Locus 可以通过脚本以语义化方式修改 Unity 场景与资产，无需直接操作磁盘上的大型 YAML 文件，也无需在大量编辑器 MCP 工具之间手动选择合适的调用方式。
- 我们面向 Unity 资产与组件设计了一套遵循「渐进式展开」思想的 Unity YAML / Inspector 语义访问层与索引视图。借助这套能力，Locus 可以在层级结构中进行导航与浏览，按需加载所需内容，而无需一次性读入并自行解析整个大型文件。
- 我们针对游戏项目构建了一整套知识系统，支持 Design / Memory / Skill / Reference 四类知识的维护与注入、对话中的知识提案、Skill 加载，以及外部文档导入（例如 Unity API Reference、飞书文档等）。该系统同时支持词法检索与语义检索，并内置本地 embedding 运行时配置与模型下载、激活流程，可运行于本地 CPU 或 GPU 环境。
- 我们基于 Rust 实现了 Unity YAML 语义解析与并行处理管线。依托这套管线，Locus 可以直接从磁盘读取并索引大型 Unity YAML 文件，并支持在版本管理流程中进行语义级别的 diff 与 merge。
- 我们基于 Rust 实现了一套高度并行的 Unity 资产数据库。基于这套数据库，Locus 可以高效搜索项目中的资产文件，并维护完整的双向依赖关系，从而支持直接查询引用关系与依赖关系。
